Overview

F-Keys turns your smartphone browser into a live macro keypad for your PC. It works entirely on your local network — no cloud, no accounts, no installed apps on the phone.

The system has three parts: a Node.js server that bridges communication, a PC dashboard where you build and manage button layouts, and a phone controller page that becomes your physical keypad.

ALPHA STATUS This is a v0.1.0 alpha build. Core functionality is complete and working. Some advanced features (cloud relay, long-press actions, multi-page layouts) are on the roadmap. Report issues via GitHub.

Quick-Start
Step 1 — Get the files

Download the three alpha files from the download section and place them in the same folder:

fkeys-server.js
fkeys-dashboard.html
fkeys-controller.html
Step 2 — Install dependencies

Open a terminal in that folder and run:

1npm install ws qrcode @nut-tree-fork/nut-js
IF NUT-JS FAILS If the install errors on @nut-tree-fork/nut-js, run npm install ws qrcode first to get the server running, then fix nut-js separately. See nut-js troubleshooting.
Step 3 — Start the server
1node fkeys-server.js

You'll see a banner with your server URLs:

╔══════════════════════════════════════════════╗
║ F-KEYS SERVER v0.1.0 RUNNING ║
╠══════════════════════════════════════════════╣
║ Dashboard: http://192.168.1.42:7331/
║ Controller: http://192.168.1.42:7331/controller
║ QR Code: http://192.168.1.42:7331/qr
╚══════════════════════════════════════════════╝
Step 4 — Open the dashboard

Open the Dashboard URL in Chrome or Edge on your PC. The status bar should show SERVER: LIVE within a second.

Step 5 — Pair your phone

On your phone: open the camera app (or a QR scanner), scan the QR code shown on the dashboard. Your phone's browser opens the controller page and connects automatically. The dashboard status bar shows CONTROLLER: LIVE.

Step 6 — Press a button

Tap any button on your phone. The dashboard live feed lights up showing the keypress, and the action fires as a real OS-level keystroke on your PC.

Config Block

Each file has a CONFIG / CFG block at the top. Modify these values to adjust behavior without digging into the logic.

Action Strings

The Action field in the key editor accepts a +-separated keystroke string. F-Keys resolves each token to an OS key via nut-js.

Format
# Single key
f5

# Key combo
ctrl+c

# Three-key combo
ctrl+shift+s

# Media / system key
mute
Supported Tokens

shiftLeft Shift

UNRECOGNIZED TOKEN? If you use a token not in the table, the server logs a warning: WARN: unresolved key token "[x]". That token is skipped but other keys in the combo still fire.
Layout JSON

Layouts are plain JSON files. Export from the dashboard, share freely, import on any F-Keys install.

Schema
{
"cols": 4, // grid columns (1–8)
"rows": 3, // grid rows (1–8)
"keys": [
{
"id": "k1", // unique string ID
"label": "COPY", // button display text
"sub": "ctrl+c", // subtitle / hint text
"action":"ctrl+c", // keystroke action string
"color": "#111820" // optional hex background
}
]
}

All fields except id, label, and action are optional. Omitting color uses the default dark background.

Dashboard Guide
Left Panel — Pairing

The server IP is auto-detected on load. If it shows the wrong IP, type the correct one and click CONNECT. The QR code updates automatically. Below the QR code, the Grid Config inputs control how many columns and rows your layout has.

Center Canvas — Layout Builder

Click any key to select it (highlighted green border). Use the Key Editor in the right panel to change its label, subtitle, action, and background color. Changes update the button in real time. Click + ADD KEY to append a new key. Click − REMOVE to delete the selected key.

Right Panel — Key Editor

Label: What's displayed on the button (large text).
Subtitle: Small hint text below the label — usually the action string or a description.
Action: The keystroke string fired when the button is pressed. See Action Strings.
Background Color: Per-key color picker. Use it to group related keys visually.

Right Panel — Live Feed

Every keypress from the phone appears here in real time with the key label, action fired, and timestamp. Useful for confirming the loop is working.

▶ PUSH TO PHONE

Sends your current layout to the phone over WebSocket. The phone re-renders its grid immediately. Push any time you make layout changes — they don't apply to the phone automatically.

Phone Controller

The controller page is a touch-optimized version of your layout. It auto-scales to fit your screen. The status badge at the top right shows connection state: LIVE (green), OFFLINE (red), CONNECTING….

Tap a button — the visual press animation fires, the phone vibrates (Android), and the keystroke is sent over WebSocket to your PC within milliseconds.

The controller auto-reconnects if it loses the WebSocket connection. No manual reload needed.

QR Pairing

The QR code encodes your controller URL: http://[your-local-ip]:7331/controller. Scan it with your phone's camera or a QR app. The browser opens automatically.

If the QR scan opens to an error, check that your firewall allows port 7331 (see Firewall / Ports). You can also navigate manually by typing the controller URL into your phone's browser.

BOOKMARK THE URL Once connected once, bookmark the controller URL on your phone. Future sessions just need the server running — no QR scan needed.

RELAY-ONLY MODE Without nut-js, the server still runs and keypresses appear in the dashboard live feed — but no OS keystrokes are fired. The server logs: Keystroke injection: INACTIVE on startup.
Firewall / Ports

F-Keys uses two ports. Both must be reachable from your phone:

PortProtocolPurpose
7331TCP (HTTP)Serves dashboard + controller pages, QR endpoint
7332TCP (WebSocket)Real-time keypress relay
Windows Defender Firewall

On first run, Windows may prompt you to allow Node.js through the firewall. Click Allow for private networks. If you missed the prompt, open Windows Defender Firewall → Allow an app → add node.exe with Private checked.

Router AP Isolation

Some routers block device-to-device traffic on the same Wi-Fi (AP isolation / client isolation). If your phone can't reach the PC, check your router settings. Alternatively, hotspot your phone directly to the PC — this bypasses the router entirely.

Changing Ports

Edit HTTP_PORT and WS_PORT in the CONFIG BLOCK at the top of fkeys-server.js. Update the matching WS_PORT and HTTP_PORT values in fkeys-dashboard.html's CFG block.

Common Errors
Dashboard shows "DISCONNECTED" immediately

The server is not running, or the IP in the dashboard connect field is wrong. Check the terminal where you ran node fkeys-server.js. If the server crashed, check for port-in-use errors and change HTTP_PORT or WS_PORT if needed.

Phone QR scan fails / "Site can't be reached"

Firewall is blocking port 7331, or the phone is on a different network. See Firewall / Ports.

Keystrokes fire on the wrong window

nut-js injects keystrokes into whichever window has OS focus. Click the window you want to control on your PC before pressing phone buttons.

fkeys-controller.html not found

The server returns this message if fkeys-controller.html is not in the same directory as fkeys-server.js. Move both files to the same folder.

fkeys-controller.html not found — make sure it is in the same folder as fkeys-server.js
Error: listen EADDRINUSE :::7331

Another process is already using port 7331. Either stop that process or change HTTP_PORT in the server CONFIG BLOCK to a free port like 7333.
